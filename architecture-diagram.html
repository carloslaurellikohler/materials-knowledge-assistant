Fase 5 — Observabilidade
Monitoramento & Otimização
📊
Amazon CloudWatch
Dashboard: token usage, latência P95, custo/query, hallucination rate. Anomaly detection para bursts. Bedrock Model Invocation Logs.
Task 4.3.1 · 4.3.2
⊕
AWS X-Ray
Distributed tracing: API Gateway → Lambda → Bedrock → OpenSearch → DynamoDB. Identifica gargalos em cada etapa do pipeline RAG.
Task 4.3.1
📋
AWS CloudTrail
Audit log de todas as chamadas Bedrock, S3, OpenSearch. Rastreabilidade completa de cada invocação de modelo. Compliance.
Task 3.3.2
⚙
ElastiCache (Redis)
Semantic caching: hash da query → resposta cacheada. Evita chamadas repetidas ao Bedrock. Reduz custo e latência em até 90% para queries recorrentes.
